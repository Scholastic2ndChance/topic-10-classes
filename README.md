# Topic 10: Classes and Exceptions

**Week 5 | CIS Intro to Programming**

---

## Overview

This week you will practice object-oriented programming and error handling — two skills that are essential in real-world Python development. You have creative freedom on the theme, but your program must demonstrate both concepts.

## Assignment Prompt

Write a Python program that defines and uses at least one class, and handles at least one exception. Your program should:

- Define a class with at least two attributes and at least one method (beyond `__init__`)
- Create at least one instance of your class and call its method(s)
- Use a `try`/`except` block to handle at least one error gracefully (e.g., invalid input, a value that doesn't exist, a division by zero)
- Print meaningful output that demonstrates both the class and the exception handling working

Some ideas: a simple bank account class, a `Student` class that calculates a GPA, an inventory item class, or a simple game character class.

## Peer Review Prompt

When reviewing a classmate's code this week, consider:

- Does their class represent a real-world thing that makes sense as an object?
- Are their attribute names and method names descriptive?
- Does their exception handling give the user a clear message about what went wrong?
- Could their `except` block be more specific (e.g., catching `ValueError` instead of bare `Exception`)?

## Notes

Focus on making your class represent something real and coherent. A class with three well-named attributes and one clear method is better than a class with many attributes that don't relate to each other.
